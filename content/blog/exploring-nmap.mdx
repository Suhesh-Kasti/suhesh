Enumeration is the first real step in any engagement. Before you exploit anything, you need to know what's there. Nmap answers that question.

<IdeaNode color="green">**Golden rule:** Don't scan targets you don't own without written permission. Network scanning without authorization is illegal in most jurisdictions.</IdeaNode>

## Your First Scans

<Accordion title="Ping sweep — find live hosts" color="#00cc55">
```bash
nmap -sn 192.168.1.0/24
```
Sends ICMP echo, TCP SYN to port 443, TCP ACK to port 80, and ICMP timestamp. Shows which IPs respond.
<CopyButton text="nmap -sn 192.168.1.0/24" label="Copy" />
</Accordion>

<Accordion title="Quick port scan" color="#ff2d95">
```bash
nmap -sS -p 1-1000 192.168.1.1
```
SYN scan on the top 1000 ports. Fast, stealthy (half-open connections).
<CopyButton text="nmap -sS -p 1-1000 192.168.1.1" label="Copy" />
</Accordion>

<Accordion title="Service version detection" color="#0055ff">
```bash
nmap -sV -p 22,80,443 192.168.1.1
```
Probes open ports to determine: service name, version number, OS hints, and sometimes hostname.
<CopyButton text="nmap -sV -p 22,80,443 192.168.1.1" label="Copy" />
</Accordion>

## Understanding Output

<Accordion title="Reading scan results" color="#7700ff">

```
PORT     STATE    SERVICE      VERSION
22/tcp   open     ssh          OpenSSH 8.9p1
80/tcp   open     http         Apache httpd 2.4.54
443/tcp  closed   https
3306/tcp filtered mysql
```

- **open** — A service is listening
- **closed** — Port accessible but no service
- **filtered** — Firewall blocking — can't tell if open or closed

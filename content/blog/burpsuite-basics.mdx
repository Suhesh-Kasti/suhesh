BurpSuite is a proxy that sits between your browser and the target. Every request passes through it. You inspect, modify, replay, and automate. It's the central nervous system of web app pentesting.

<IdeaNode color="blue">**Free (Community) vs Pro:** Community has Proxy, Repeater, Intruder (throttled), Decoder. Pro adds Scanner (automated vuln detection). For most work, Community is enough.</IdeaNode>

## The Core Tools

<Accordion title="Proxy — Intercept traffic" color="#0055ff">

Burp acts as a man-in-the-middle between browser and server. Configure your browser to use `127.0.0.1:8080`, install Burp's CA certificate, and every HTTP request flows through the Proxy tab.

- **Intercept** — pause and modify requests before they reach the server
- **HTTP History** — every request/response logged for analysis
- **Scope** — limit to target domain to reduce noise

</Accordion>

<Accordion title="Repeater — Manual testing" color="#00cc55">

Send a request, modify it, send again. Repeat. The bread and butter of manual testing.

```http
POST /login HTTP/1.1
Host: target.com
Content-Type: application/x-www-form-urlencoded

username=admin' OR '1'='1&password=anything
```

</Accordion>

<Accordion title="Intruder — Automate attacks" color="#ff6600">

Automated fuzzing. Select positions in a request, load payloads, and fire. Used for:
- Brute-forcing passwords
- Fuzzing parameters for SQLi/XSS
- Enumerating valid usernames
- Testing rate limiting

</Accordion>

<Accordion title="Decoder — Encode/decode" color="#7700ff">

Base64, URL encoding, HTML entities, hex, etc. Right-click any selected text in Burp → "Send to Decoder."

</Accordion>

Your password manager tells you "this password appeared in 2847 breaches" — instantly. How? It doesn't have a breach database on your device. And it definitely shouldn't send your password to a server. So what's the trick?

<IdeaNode color="pink">**k-anonymity with hash prefixes.** Send the first 5 characters of a SHA-1 hash. Get back all matching suffixes. Compare locally. Your actual password never touches the network.</IdeaNode>

## The 3-step flow

1. **Hash** your password with SHA-1 → `6c7ca345f63f835cb353ff15bd6c5e052ec08e7a`
2. **Send only the first 5 chars:** `6c7ca` to the API
3. **Receive all suffixes** that start with `6c7ca` — compare locally

<Accordion title="Walked example: password 'mypassword123'" color="#ff2d95">

```
SHA-1("mypassword123")
→ 6c7ca345f63f835cb353ff15bd6c5e052ec08e7a

Send to server: "6c7ca"
Server returns: 500+ hash suffixes + breach counts
  345F63F835CB353FF15BD6C5E052EC08E7A:2847  ← Match!
  891A2F3B4C5D6E7F8A9B0C1D2E3F4A5B6C7D8E9F:156
  ...

Compare locally: suffix matches → password breached 2847 times
```

</Accordion>

## Why 5 characters?

- **16^5 = ~1 million possible prefixes** — each prefix covers hundreds of hashes, providing anonymity
- **O(1) lookup** — the server indexes by prefix, no scanning needed
- **Network-friendly** — returns ~500 results per query, sub-100ms response

<InteractiveCode defaultLanguage="python" defaultValue={`import hashlib
import requests

def check_breach(password):
    sha1 = hashlib.sha1(password.encode()).hexdigest().upper()
    prefix, suffix = sha1[:5], sha1[5:]
    
    resp = requests.get(
        f"https://api.pwnedpasswords.com/range/{prefix}"
    )
    
    for line in resp.text.splitlines():
        h, count = line.split(":")
        if h == suffix:
            return f"Breached {count} times!"
    return "Not found in any known breaches"

# Try it
print(check_breach("password123"))
`} />

<IdeaNode color="green">Your password never leaves your device. The server sees a 5-char hash prefix out of a 40-char hash — impossible to reverse to the original password.</IdeaNode>

## Why this is brilliant

**Privacy:** 5 hex chars of a hash reveal nothing about the original password. The server can't even tell WHICH hash you're checking — it returns all of them in that prefix bucket.

**Speed:** Pre-sorted indexed database. Direct lookup, no scanning through millions of records. The response is ~500 lines, transfer size under 20KB.

**No local database needed:** You don't need to download hundreds of millions of breached password hashes to your device. The server does the heavy lifting; your device just checks one hash.
